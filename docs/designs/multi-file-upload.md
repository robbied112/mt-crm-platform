## Vision

### 10x Check
A batch import command center — drop 10 files, they all auto-parse and auto-map simultaneously, see a unified table of all files with detected types and row counts, approve/reject each mapping at a glance, one "Import All" button. Progress bar per file. Full undo per file.

### Current Plan (Phase 1)
Multi-file upload with sequential processing. Drop N files, auto-confirm high-confidence mappings, pause for manual review on low-confidence, batch completion summary.

## Accepted Scope
- Multi-file upload via drag-drop and file picker (max 20 files)
- File queue with sequential processing through existing pipeline
- Per-file type auto-detection badges (QuickBooks, Product Sheet, etc.)
- Progress indicator showing current file / total
- Duplicate file detection (warn on same filename in last 100 uploads)
- Remove-from-queue x button (queued files only)
- Batch completion summary with file counts and total rows
- Auto-confirm high-confidence mappings, pause for low-confidence review
- Skip failed files, continue processing queue
- useFileQueue() custom hook for state management
- batchId/batchIndex in upload log for observability

## Architecture

```
DataImport.jsx (orchestrator)
       |
  useFileQueue() hook
  [{file, status, type, parsed, mapping, conf, result, error}]
       |
  +---------+----------+-----------+
  |         |          |           |
Queue UI  Active    Batch Done  loadUploads()
(list,    File      (summary,   (dup check,
badges,   (mapping/  per-file    last 100)
x remove, preview/   results)
progress) confirm)
```

## Key Design Decisions
1. Sequential processing (not parallel) — avoids Firestore race conditions
2. High-confidence auto-confirm — AI mapper confident? Skip mapping step
3. Failed files skip, don't block queue — one bad file doesn't ruin the batch
4. Queue cap at 20 files — prevents browser freeze on large drops
5. Duplicate check limited to last 100 uploads — fast query, practical coverage
