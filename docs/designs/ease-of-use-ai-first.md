## Problem Statement

CruFolio was built data-out (structured distributor reports as input), not user-in. The result:
- 20 routes, most empty on day 1
- Upload buried in Settings > Data Upload
- AI (Claude Sonnet comprehension) is genuinely smart but invisible to users
- A small winery owner feels overwhelmed, not empowered

**The competitive question:** Is CruFolio easier than just pasting a CSV into ChatGPT and asking "which accounts are growing?" If not, we lose.

## Vision

### 10x Check
CruFolio doesn't have a sidebar with 20 empty pages. It has ONE screen when you sign up: a file drop zone with conversational AI. You drop your file, the AI says "This looks like a Southern Glazer's depletion report for California — I see 2,400 rows across 47 accounts. Here's what I found..." and immediately shows a mini-dashboard preview RIGHT THERE. No Settings. No mapping review. No navigation. Just insight.

Each subsequent file adds more intelligence. "Now that I have your depletions AND your inventory data, I can calculate days-on-hand and predict reorders." The app grows WITH the user's data, not ahead of it.

### Platonic Ideal
The interface is almost invisible. No complex navigation. No configuration. The app infers your role from your data (depletion reports → you're a supplier). It infers your distributors from your files. It builds your product catalog from your SKU columns automatically.

**The feeling:** Like having a brilliant, detail-oriented analyst who already knows the wine trade and never forgets anything.

- **First 30 seconds:** "I dropped a file and it already knows what it is"
- **First 2 minutes:** "I can see my top accounts and which ones need attention"
- **First week:** "It reminded me that Bevmo hasn't reordered in 45 days"
- **First month:** "I can't imagine going back to spreadsheets"

## Architecture

### Key Decisions

1. **New UploadContext** — Shared provider for file queue state, learned mappings cache, analysis timing. Separate from DataContext (which is the read model for dashboards).

2. **Shared UploadFlow component** — Extracted from DataImport/index.jsx. Used by BOTH the upload homepage and Settings > Data Upload. DRY.

3. **Client-side transforms for preview** — `transformData.js` (already in `packages/pipeline/src/`) runs in the browser on a 500-row sample. Preview is best-effort with try/catch fallback.

4. **Learned mappings** — `tenants/{tenantId}/config/learnedMappings` — flat Firestore doc. Capped at 200 entries with LRU eviction.

5. **Insight narration** — New `generateInsightNarration` in `comprehend.js`. Claude Sonnet, ~2K tokens. Cached in `views/_narration`, regenerated only on new import.

6. **Feature flags** — Each proposal independently toggleable via `tenantConfig.features.*`.

### Upload-First Data Flow

```
  USER DROPS FILE
       │
       ▼
  parseFile() ── nil/empty/too-large? → clear error message
       │
       ▼
  comprehendReport() ── fail? → conversational recovery (not silent fallback)
       │
       ▼
  AUTO-CONFIRM CHECK (95%+ target with learned mappings)
       │
  ┌────┴────────────────────┐
  │ High confidence          │ Low confidence
  ▼                          ▼
  transformData(500 rows)   CONVERSATIONAL RECOVERY
  → InstantPreview           "Can you help with 2 columns?"
  → SpeedBadge               → User fixes → learned mapping saved
       │                     → same preview flow
       ▼
  "Save & See Full Dashboard"
       │
       ▼
  SAVE → rebuild → generateInsightNarration
       │
       ▼
  DASHBOARD + narration card + "What Should I Upload Next?"
  + Progressive sidebar reveals new routes
```

## Competitive Moat Analysis

**Where CruFolio wins over ChatGPT after this ships:**
- Persistent memory (knows your accounts, history, trends over time)
- Visual dashboards (charts, scorecards, pipeline views)
- Automated alerts (Daily Actions, reorder predictions)
- Team features (multiple users, roles, shared data)
- Industry-specific intelligence (wine trade terminology, distributor patterns)
- Gets smarter per tenant (learned mappings, correction memory)

**Remaining gap (Phase 2):**
- Ad-hoc questions → TODO-121 "Ask CruFolio" chat interface

**Long-term moat:** Accumulated intelligence. CruFolio gets smarter with every file, every correction, every interaction. ChatGPT starts fresh every conversation.
